## Get Order Details (Customer)

<api method="GET" path="/api/orders/{orderId}/details" auth="Customer" />

Retrieves complete details for a specific order.

### Path Parameters

<ParamField path="orderId" type="string (uuid)" required>
  Order identifier
</ParamField>

### Response

<ResponseField name="id" type="string (uuid)">
  Order identifier
</ResponseField>

<ResponseField name="orderNumber" type="string">
  Human-readable order number
</ResponseField>

<ResponseField name="status" type="string">
  Order status (Pending, Processing, Shipped, Delivered, Cancelled)
</ResponseField>

<ResponseField name="createdAt" type="string (datetime)">
  Order creation timestamp
</ResponseField>

<ResponseField name="totalAmount" type="number">
  Total order amount
</ResponseField>

<ResponseField name="deliveryAddress" type="object">
  Delivery address details
</ResponseField>

<ResponseField name="deliveryMethod" type="object">
  Delivery method information
</ResponseField>

<ResponseField name="items" type="array">
  Array of order items with product details
</ResponseField>

<ResponseField name="paymentStatus" type="string">
  Payment status (Pending, Paid, Failed, Refunded)
</ResponseField>

### Response Codes

- `200 OK` - Order details retrieved
- `404 Not Found` - Order not found

### Examples

<CodeGroup>

```bash cURL
curl -X GET "https://your-server.com/api/orders/order-uuid/details" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  `https://your-server.com/api/orders/${orderId}/details`,
  {
    headers: {
      'Authorization': `Bearer ${jwtToken}`
    }
  }
);

const orderDetails = await response.json();
console.log('Order:', orderDetails);
```

```python Python
import requests

response = requests.get(
    f'https://your-server.com/api/orders/{order_id}/details',
    headers={'Authorization': f'Bearer {jwt_token}'}
)

order_details = response.json()
print(f"Order Number: {order_details['orderNumber']}")
print(f"Status: {order_details['status']}")
print(f"Total: ${order_details['totalAmount']}")
```

</CodeGroup>

## Get Customer Orders (Customer)

<api method="GET" path="/api/orders/{customerId}" auth="Customer" />

Retrieves all orders for a specific customer.

### Path Parameters

<ParamField path="customerId" type="string (uuid)" required>
  Customer identifier (must match authenticated user)
</ParamField>

### Response

<ResponseField name="orders" type="array">
  Array of customer order objects
</ResponseField>

### Response Codes

- `200 OK` - Orders retrieved
- `404 Not Found` - Customer not found

### Examples

<CodeGroup>

```bash cURL
curl -X GET "https://your-server.com/api/orders/customer-uuid" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  `https://your-server.com/api/orders/${customerId}`,
  {
    headers: {
      'Authorization': `Bearer ${jwtToken}`
    }
  }
);

const orders = await response.json();
console.log(`Found ${orders.length} orders`);
```

```python Python
import requests

response = requests.get(
    f'https://your-server.com/api/orders/{customer_id}',
    headers={'Authorization': f'Bearer {jwt_token}'}
)

orders = response.json()
print(f"Found {len(orders)} orders")
```

## Get Seller Orders (Seller)

<api method="GET" path="/api/orders/sellers/{sellerId}" auth="Seller" />

Retrieves all order items for products belonging to a specific seller.

### Path Parameters

<ParamField path="sellerId" type="string (uuid)" required>
  Seller identifier (must match authenticated seller)
</ParamField>

### Response

<ResponseField name="orderItems" type="array">
  Array of order items for this seller's products
</ResponseField>

### Response Codes

- `200 OK` - Order items retrieved
- `404 Not Found` - Seller not found

## Order Status Values

Orders progress through the following statuses:

- **Pending** - Order created, awaiting payment confirmation
- **Processing** - Payment confirmed, order being prepared
- **Shipped** - Order has been shipped to customer
- **Delivered** - Order delivered to customer
- **Cancelled** - Order cancelled by customer or seller
- **Refunded** - Order refunded to customer

## Payment Status Values

- **Pending** - Payment not yet completed
- **Paid** - Payment successfully received
- **Failed** - Payment attempt failed
- **Refunded** - Payment refunded to customer

## Order Management Tips

### For Customers

<Info>
  Track your orders in real-time by checking order status regularly. You'll receive notifications for status updates.
</Info>

### For Sellers

<Info>
  Process orders promptly to maintain good customer satisfaction. Update order status as items are prepared and shipped.
</Info>

<Warning>
  Sellers are responsible for updating order status and providing tracking information to customers.
</Warning>
